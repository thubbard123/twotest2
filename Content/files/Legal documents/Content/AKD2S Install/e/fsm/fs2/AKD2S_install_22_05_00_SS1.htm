SS1-t (Safe Stop 1)

SS1-t (time controlled) description for drive option Functional Safety 2.

Description

With the time based function "Safe Stop 1" (SS1-t) the drive is stopped by a controlled braking with the configured emergency brake ramp.

When the time t_SS1 (AXIS#.SAFEPARAM.SS1_#.TIMETOSTO) is elapsed, STO is activated.

SS1 function corresponds to a controlled braking according to IEC 60204-1, stop category 1.

The controlled braking is no functional safe process. The STO function which is activated when time t_SS1 is elapsed, is a safe function.

Number of instances

Three instances per axis.

Function Input/Output variables

Inputs

  • SS1 Activate can be triggered from
  • FSoE , or
  • I/O failure (instance 1 only), or
  • safe digital inputs (must be mapped).
  • STO Activate is triggered from SS1.
  • STO Restart Acknowledge is used for manual restart (see STO (➜ # 1)).
    The signal can be triggered from
  • FSoE, or
  • safe digital inputs (must be mapped).

Outputs

  • SS1 Active, logical status of the SS1 function
  • SS1 State, logical status of the digital SS1 inputs (not mappable to SOUT)
  • STO Active, logical status of the STO function
  • STO Restart Request is used for manual restart (see STO (➜ # 1)). The signal can be evaluated by using FSoE or safe digital outputs (must be mapped).

 

Activation

The safety function can be mapped with or by using FSoE to the safe digital inputs or other functions.

If SS1_2 or SS1_3 is set to “Not Used”, the corresponding FSoE activation bit is ignored even if SAFEFIELDBUS.ACTIVATION is 1.

Activation by FSoE

For drive profile overview refer to (➜ # 1). For setup and handling refer to the EtherCAT Manual.

Activation by safe digital inputs

Technical data of the safe inputs (➜ # 1). If activation by safe digital inputs is required, then the digital SS1 inputs can be connected

  • to transistor outputs of a safety control, or
  • to relay outputs of a safety control, or
  • to an emergency stop switch.

If an SS1 input goes open-circuit or 0 V, then motor ramps down for the configured time T. After a delay or triggered by a safe input the SBC, SBD and STO functions become active if configured.

Review the enclosure and wiring instructions (➜ # 1).

Example: SS1, dual channel, 2 axes, SIL3 PLe, Safety Control, electronic outputs, automatic restart, DISTINCTPULSE must be deactivated.

Example: SS1, dual channel, 2 axes, SIL3 PLe, Safety Control, relay contacts, automatic restart, DISTINCTPULSE must be deactivated.

Safety level must be evaluated referring to B10d value of the relay contacts.

 

Example: SS1, dual channel, 2 axes, SIL3 PLe, Emergency Stop, automatic restart, DISTINCTPULSE must be deactivated.

Safety level must be evaluated referring to B10d value of the emergency stop contacts.

Example: SS1 dual channel input, one axis, SIL3 PLe, Emergency Stop, manual restart, DISTINCTPULSE must be deactivated.

Safety level must be evaluated referring to B10d value of the emergency stop contacts.

Restart

Restart can be set as "automatic restart" or "manual restart" (STO Restart Mode, AXIS#.SAFEPARAM.STO.RESTARTACKMODE).

  • With manual restart a restart request signal will be send to the safety control, when STO is deactivated. The axis waits for the restart acknowledge and axis enable to restart.
  • With automatic restart the axis restarts when STO is deactivated and axis is enabled.

Setup in Workbench

Select the drive and open the Safety Parameterization view (➜ # 1).

Choose the axis (1), select category (2), SS1-t function (3) and instance number (4).

The safety parameters are displayed and can be configured.

If one dual input (e.g. SIN1/SIN2) shall activate SS1-t for both axis 1 and axis 2, then set the Activation parameter for both axes to the identical input (e.g. Safe Dual Input 1).

Fault Reaction/ Failure Messages

If dual channel mode is selected, the drive supervises if the two inputs are in a different state for more than 100 ms. If the two inputs are in different state for more than 100 ms, an I/O failure activates SS1_1 and then the axis is switched off by STO.

An I/O failure warning is messaged when the drive is disabled or a fault when the drive is enabled and AXIS#.SAFE.STO.REPORTFAULT is set to 1.

Safety State/ Status signals

The signal AXIS#.SAFE.SS1#.INTERNALSTATE can be monitored by safe digital outputs with OSSD pulses (➜ # 1). The status signal must be mapped to the safe digital output. Two outputs can be combined to a dual channel output. For parameter description refer to (➜ # 1).
