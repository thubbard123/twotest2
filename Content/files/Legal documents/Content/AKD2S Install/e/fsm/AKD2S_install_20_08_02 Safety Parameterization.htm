Parameters Editor

Common safety parameters

For general safety settings select "Common" and "General".

The SafeID (set with the rotary switches on the drive) is visible in the first line.

 

For safe input setting select "Common" and "Inputs"

Define the required behavior for OSSD test pulses (➜ # 1). If you use dual channel inputs for a safety function, select the used safe input pair. For parameter description refer to (➜ # 1).

For safe output setting select "Common" and "Outputs"

If you need dual channel outputs for a status message, select the used safe output pair.
For parameter description refer to (➜ # 1).

 

Axis safety parameters

Choose the axis (1), select category (2), safety function (3) and instance number (4) .

Parameterization see description of the specific safety function (➜ # 1) (➜ # 1). For parameter description refer to (➜ # 1).

Define a parameterization package

When all parameters for all safety functions and for all devices are set, you can finish the parameterization. You must define a parameter bundle called "Parameterization Package".

Make sure you are on the device topology “Safety”, click on your Safety Group name, and then on the "Safety Parameters" tab at the top. Then click on “new package” button.

The safety parameters are bundled to a package in the “Parameterization Packages” screen.

View / Export / Delete a parameterization package

To see the contents of a parameter package, select the according package (1), click on the arrow next to “View” (2) and select “View Parameter Report” (3).

To save a parameter package externally click on “Export” (4).

To delete this package, click on “Delete” (4).
